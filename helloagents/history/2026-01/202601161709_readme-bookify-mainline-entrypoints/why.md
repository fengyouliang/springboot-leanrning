# Why：让仓库“更像书”（README 入口 + 主线章节可跑入口）

## 背景

本仓库已经具备“多模块学习 + 文档站点（MkDocs）+ Book-only 主线之书”的形态，但读者在进入仓库时仍可能遇到两类摩擦：

- 根 `README.md` 第一屏更偏“模块导航”，读者不一定立刻意识到“可以按章节顺读”。
- 主线之书（0–18 章）虽已串起叙事主线，但缺少统一、明确的“本章最小可跑入口（命令）+ Exercise 入口”，导致读者需要回到模块 docs 或自行搜索测试类。

## 目标

- 在根 `README.md` 前置“书籍化入口”：两条阅读路径（顺读/目标驱动）+ 主线目录入口（0–18 章）。
- 在 Book-only 主线之书的 0–18 章中，为每章补齐可跑闭环：
  - 最小可跑入口：对应 `*LabTest` + Maven 运行命令
  - 动手练习入口：对应 `*ExerciseTest`（默认 `@Disabled`）

## 非目标

- 不改写各模块 docs 正文内容与章节结构（避免大规模迁移与断链）。
- 不新增/修改 Java 代码与测试逻辑（仅补齐入口信息与可跑提示）。
