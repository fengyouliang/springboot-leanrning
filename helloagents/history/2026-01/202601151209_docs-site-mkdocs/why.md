# Why：文档站点（MkDocs）落地

## 背景

本仓库的文档已具备“模块内 book style（`docs/README.md` 为目录页）+ 可运行 Labs/Exercises 证据链”的特征，但阅读体验仍偏“仓库内 Markdown 散落”：

- 缺少统一的全文搜索与跨模块统一导航
- 读者需要在 GitHub/IDE 中跳转阅读，难以形成“像书一样”的连贯体验
- 作为“技术专栏合集”风格，文章本身应该保持叙事流；结构化索引与串联更适合由站点层承接

## 目标

- 把仓库内多模块 `docs/` 聚合为一个静态站点（可本地预览、可 CI 构建发布）
- 侧边栏按主题顺序组织（主线优先），同时保留模块内目录页作为 SSOT
- 不强制每篇文章采用固定“章节契约”写法；站点提供搜索/导航/索引即可

## 非目标

- 不改写现有模块 docs 正文内容与目录结构（避免大规模迁移导致断链）
- 不在本次引入复杂的组件化站点（例如 Docusaurus 的 React 页面重写）
